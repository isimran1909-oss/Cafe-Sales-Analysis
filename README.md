#  Cafe Sales Analysis using Python & Power BI

##  Project Overview

This project demonstrates an end-to-end data analysis workflow on a Cafe Sales dataset using Python and Power BI. The objective is to clean raw transactional data, perform exploratory data analysis (EDA), build meaningful visualizations, and generate business insights to support data-driven decision-making.

---

##  Objectives

- Clean and preprocess raw sales data
- Handle missing and invalid values
- Perform Exploratory Data Analysis (EDA)
- Visualize customer purchasing behavior
- Build an interactive Power BI dashboard
- Generate actionable business insights

##  Data Cleaning

The dataset was cleaned using Python by:

- Converting numeric columns
- Handling missing values
- Replacing invalid entries (UNKNOWN, ERROR)
- Formatting transaction dates
- Creating a cleaned dataset for analysis

---

##  Visualizations

The project includes:

- Quantity Distribution
- Total Sales by Location
- Payment Method Distribution
- Price vs Quantity Analysis
- Sales Trend Over Time

---

##  Power BI Dashboard

The interactive dashboard provides insights into:

- Total Sales
- Sales by Location
- Payment Method Analysis
- Sales Trends
- Customer Purchasing Patterns

---

##  Key Insights

- Takeaway orders generated higher revenue than In-store orders.
- Digital Wallet was the most frequently used payment method.
- Sales remained relatively stable with one noticeable peak.
- Most purchases consisted of 2–4 items per transaction.
